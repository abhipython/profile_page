Cisco Nexus 1000v software switch

April 2015 – September 2016

Role : Engineer - Test/QA

Team size : 15

Project Description:

The Nexus 1000v is a virtual switch for use in virtual environments including both VMware vSphere, KVM and Microsoft Hyper-V. It is as such not a physical box but a software application that interacts with the hypervisor so you can virtualize the networking environment and be able to configure your system as if all virtual servers have connections to a physical switch and include the capabilities that a switch offers.

Know more about our product

My Roles and Responsibilities:
  • Set up testbed in the lab with different topologies according to feature specs and validate the setup on different platforms.
  • Configuring different switching protocols such as Port-Profile, Port-Channel, Port-Security, ACL, CTS, IGMP Snooping, QOS, PVLAN, IPSG, Span-Erspan etc. in an end to end network solution for ESXi platform.
  • Verifying the configuration by generating different packets such as TCP, UDP, IPv4, IPv6 etc. using Ostinato and Wireshark.
  • Automating the feature test using Python in parallel with the developers’ design and writing test plans in TIMs and EDCS.
  • Running basic sanity, extended sanity and regressions for Nexus 1000v features on ESXi, KVM and Hyper-V platforms.
  • Debugging the issues found by reviewing the design with the developers’ and raising defects using CDETS and PRRQ.
  • Patch testing (software support testing) by manually and by automation scripts between VMware products and Cisco Nexus 1000v.
  • Performed migration testing of supervisor module to vmware distributed switch with migration of vmnic, vswitch.
